CASE STUDY

NEB Gyan — Educational Platform

Want a quick walkthrough?
Request a demo

From scattered prep resources to a reliable learning workflow

A production platform for +2 learners: content, practice, and feedback — designed to hold up during peak exam demand.

Users
10,000+ active
Role
Founder • Full-stack
Focus
Scale + analytics

Problem

Why it mattered
  • Students relied on scattered notes, groups, and PDFs with uneven quality.
  • Mock tests existed, but learners couldn’t see progress or weak areas.
  • Many resources broke under traffic during exam season.

Solution

What I built
  • Structured notes + timed mock tests + a clear dashboard.
  • Topic-level performance signals to highlight weak areas.
  • Reliable backend patterns: auth, caching, and DB optimization.

Architecture

How it works
  • Django + DRF for modular services and clean API boundaries.
  • MySQL for core data; Redis for caching and performance.
  • Analytics events modeled for progress tracking and reporting.

UX decisions

Clarity first
  • Make the loop obvious: learn → practice → feedback → next action.
  • Reduce cognitive load: consistent test flow, clean result screens.
  • Mobile-first constraints (most learners use phones).

Outcomes

Impact + next
  • Scaled to 10,000+ active users with stable peak performance.
  • Next: privacy-first analytics docs (minimization + retention) and teacher dashboards.

CASE STUDY

Adaptive AI Tutor (Prototype)

Prototype repository
Open GitHub

Mastery-aware practice sequencing

Exploring how to model what a learner knows and adapt practice using learning-science-inspired ML.

Type
Research prototype
Methods
DKT • bandits • CodeBERT
Goal
Adaptive tutoring loop

Problem

Learning gap

Most practice systems treat all learners the same. In reality, mastery changes over time and by concept—especially in programming.

  • Practice can be too easy (waste time) or too hard (discourage).
  • Without mastery modeling, recommendations are guesswork.

Approach

Modeling + policy
  • Deep Knowledge Tracing (LSTM): predict mastery from interaction sequences.
  • Contextual bandits: select next practice item to balance learning + exploration.
  • CodeBERT exploration: code-aware representations for programming tasks.

Evaluation

What I measure
  • Baseline comparisons, error analysis by topic, and calibration checks.
  • Repeatable pipeline: dataset prep → training → evaluation → iteration.
  • Design constraints: logs minimal + teacher oversight possible.

Next steps

Research-ready
  • Human-in-the-loop review for recommendations.
  • Privacy-aware logging with auditable events.
  • More classroom-like tasks and stronger baselines.
