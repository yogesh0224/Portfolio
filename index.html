Case study

NEB Gyan — Educational Platform

From scattered materials to a reliable learning workflow

A production platform for +2 learners: content, practice, and feedback — designed to hold up during peak exam demand.

Users
10,000+ active
Role
Founder • Full-stack
Focus
Scale + analytics

Problem Why it mattered

  • Prep content was scattered across groups, PDFs, and inconsistent sources.
  • Students lacked visibility into progress, weak topics, and next best actions.
  • Peak exam traffic caused many learning sites to slow down or fail.

Solution What I built

  • Structured notes + timed mock tests + progress dashboards.
  • Topic-level performance signals to surface weaknesses early.
  • Backend built for stability: caching, DB indexing, optimized endpoints.

Architecture How it works

  • Django + DRF for modular backend services and clean API boundaries.
  • MySQL as primary datastore; Redis for caching + performance.
  • Analytics events modeled for measurable progress and reporting.

UX decisions User-first

  • Clear loop: learn → practice → feedback → next action.
  • Low cognitive load: consistent navigation + predictable test flow.
  • Mobile-first patterns (many learners primarily use phones).

Outcome & next Impact

  • Scaled to 10,000+ active users with stable performance during peaks.
  • Next: privacy-first analytics docs, teacher dashboards, and learning impact measurement.

Adaptive AI Tutor — Prototype

A prototype for mastery-aware practice sequencing

Exploring how to model what a learner knows and adapt practice using learning-science-inspired ML.

Type
Research prototype
Methods
DKT • bandits • CodeBERT
Goal
Adaptive loop

Problem Learning gap

Most practice systems treat all learners the same. In reality, mastery shifts over time and by concept—especially in programming.

  • Too-easy practice wastes time; too-hard practice kills motivation.
  • Without mastery modeling, recommendations become guesswork.

Approach Modeling

  • Deep Knowledge Tracing (LSTM): predict mastery from interaction sequences.
  • Contextual bandits: choose next item to balance learning + exploration.
  • CodeBERT: explore code-aware representations for programming tasks.

What it demonstrates Workflow

  • Dataset prep → training → evaluation → iteration (repeatable pipeline).
  • Baseline comparisons + error analysis + calibration checks.
  • Safety-aware: tutoring logic can stay teacher-supervised.

Next iteration Research-ready

  • Teacher-in-the-loop review of recommendations.
  • Privacy-aware logging (minimal, auditable events).
  • Stronger evaluation design with classroom-like tasks + robust baselines.
This is a prototype; I can share notes on evaluation + next steps.
