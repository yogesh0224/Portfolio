NEB Gyan — Mini case study

Context

NEB Gyan is a free learning platform for +2 students and entrance prep. I founded and built it end-to-end, focusing on reliability, performance, and a simple learning workflow.

Problem
  • Prep materials were scattered across sources and formats.
  • Students lacked visibility into progress and weak areas.
  • Systems often failed under load during peak exam periods.
What I built
  • Structured content (notes) + timed mock tests + progress dashboards.
  • Weak-area identification (topic-level performance signals).
  • REST APIs, auth, caching, and database optimization for scale.
Architecture
  • Django + DRF backend with modular apps and clear API boundaries.
  • MySQL for core data; Redis for caching/session performance.
  • Analytics events modeled for progress tracking and reporting.
Outcome
  • 10,000+ active users with stable performance at peak usage.
  • Clear learning loop: practice → feedback → next actions.
Next iteration
  • Privacy-first analytics documentation (data minimization + retention policies).
  • Teacher dashboards and cohort-level insights.
  • Experimentation framework for measuring learning impact.

Adaptive AI Tutor — Mini case study

Context

A research prototype exploring how to adapt practice to a learner’s mastery—especially for programming education.

Approach
  • Deep Knowledge Tracing (LSTM) for mastery prediction over sequences.
  • Contextual bandits for adaptive practice selection policies.
  • CodeBERT exploration for code-aware representations.
What it demonstrates
  • End-to-end workflow: dataset prep → model training → evaluation → iteration.
  • Clear metrics for improvement (accuracy, calibration, error analysis by topic).
  • How tutoring logic can be constrained for safety + teacher oversight.
Next iteration
  • Human-in-the-loop feedback: teachers can review model recommendations.
  • Privacy-aware logging (minimal, auditable event data).
  • Better evaluation with classroom-like tasks and more robust baselines.
